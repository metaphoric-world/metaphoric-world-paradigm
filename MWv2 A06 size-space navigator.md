
### Appendix 6 - The size‑space navigator: clothing sizes as a semantic field

This Appendix (as well as Appendix 5) sketches an application example for the implementation architecture of the Metaphoric World (Chapter 25).

Clothing sizes are seemingly simple labels - "M", "38", "12", "XL", and so on.
But anyone who has ever ordered clothes online knows the truth: these labels behave less like measurements and more like weather systems. They drift across regions, brands, decades, and fashion cycles. A "Medium" in Tokyo is a "Small" in Berlin; a "38" from 2005 is a "36" today; "Oversized" is not a size at all, but a cultural mood.

In other words: clothing sizes are not numbers - they are semantic fields.
And once you see them that way, the chaos becomes navigable.

1. The problem: a global drift‑system

Clothing sizes are shaped by overlapping forces:

- regional body norms  
- brand‑specific cut philosophies  
- fashion cycles (slim, relaxed, oversized)  
- historical drift (vanity sizing, shifting baselines)  
- individual body proportions  

These forces do not align.  
They create semantic drift zones — regions where the meaning of a size becomes unstable.

From the perspective of the Metaphoric World, this is not a bug.  
It is a field phenomenon.

2. The Insight: Sizes Form a Semantic Color Space

Instead of treating sizes as fixed categories, the Size‑Space Navigator treats them as positions in a semantic color space:

- each size label has a core meaning (F)  
- each has an uncertainty radius (Δ)  
- each is shaped by associative lines (A) — e.g., “slim”, “relaxed”, “boxy”  
- each is embedded in context layers (C) — region, brand, trend  
- and each has a coherence profile (K) — how stable its meaning is across contexts  

This is the same architecture introduced in Chapter 25, now applied to a domain where people feel the drift every day.

The result is not a table of sizes, but a navigable field.

3. How the Navigator Works (Narrative Overview)

The app does not calculate a “true size”.  
It constructs a personal semantic zone.

It does this by combining:

- a lightweight body scan (for proportions, not measurements)  
- brand‑specific cut profiles  
- regional drift curves  
- style preferences (“loose”, “fitted”, “structured”, “fluid”)  
- historical data (how a brand’s sizes have shifted over time)  

From these inputs, the system identifies the user’s stable region in the size‑space — the area where meaning, cut, and body shape align.

It then highlights:

- coherent zones (likely to fit)  
- drift zones (unstable meaning)  
- contextual distortions (e.g., “this brand runs small”)  
- semantic attractors (your personal style tendencies)  

The user does not receive a number.  
They receive a map.

4. Why This Works

Because the problem is not numerical.  
It is semantic.

Sizes fail not because they are inaccurate, but because they are:

- context‑dependent  
- historically drifting  
- culturally coded  
- affectively charged  
- brand‑specific  
- shape‑dependent  

This is exactly the kind of system the MW is designed to model:  
a field with stable regions, transition zones, drift, and contextual attractors.

The Navigator simply applies the same conceptual machinery as the McGyver example — but to a domain where the “materials” are meanings rather than wood and metal.

5. Why This Example Matters

It shows that the MW is not limited to:

- ecology  
- cognition  
- physics  
- semantics  
- world‑formation  

It also applies to the most mundane, everyday problems —  
problems that resist linear logic precisely because they are semantic fields.

The Size‑Space Navigator demonstrates:

- how drift becomes structure  
- how context becomes meaning  
- how coherence becomes guidance  
- how a field model can replace a broken categorical system  

And it does so in a domain where millions of people experience the instability daily.

6. Closing Note

This example is intentionally modest.  
It is not about fashion.  
It is about how meaning behaves — and how a field‑based model can turn ambiguity into orientation.

It stands alongside the McGyver example as a second demonstration of the same principle:

> When the world refuses to behave like a list of categories,  
> a field model is not optional — it is necessary.
